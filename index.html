System Overview

I design end-to-end data systems that transform unstructured documents, APIs, and transactional data into analytics-ready and AI-ready assets.

My work blends data engineering, dimensional modeling, and retrieval-augmented generation (RAG) so stakeholders can ask better questions, trust the answers, and act without digging into raw data or pipeline code.

I treat data platforms as products—with validation, observability, and iteration— not one-off scripts.

Featured Data Systems

End-to-end platforms: ingestion → modeling → analytics/AI → trust.

ORBIT AI-50 Intelligence

LLM-powered intelligence hub for Forbes AI-50 companies.

  • Objective: Automate due diligence and profiles with unified data + LLM layer.
  • Approach: Dual pipelines (structured extraction + RAG), orchestrated via Airflow; vector indexing with Qdrant.
  • Impact: Reduced manual research and improved consistency for competitive analysis.

Tools: Python, Airflow, FastAPI, Streamlit, Qdrant, ChromaDB, GCP

Aurelia Financial RAG

Cloud-native RAG microservice over a 4,000-page financial toolbox.

  • Objective: Make dense documentation searchable, explainable, and reliable.
  • Approach: Code-aware chunking, embeddings in ChromaDB, Instructor-based structured responses with citations.
  • Impact: Faster lookup and higher trust through cited, structured answers.

Tools: Python, FastAPI, Airflow, ChromaDB, Cloud Run, GCP

Report Intelligence (SEC Filings)

Document-processing backbone for SEC filings and financial RAG.

  • Objective: Reliable extraction + validation layer for filings (PDF/XBRL).
  • Approach: Docling + pdfplumber extraction, modular chunking/validation, DVC versioning for auditability.
  • Impact: Standardized noisy filings into consistent downstream assets.

Tools: Python, Docling, pdfplumber, DVC, GCP

Dow-30 IR Scraper

Central investor relations document index for the Dow-30.

  • Objective: Centralize heterogeneous IR docs for research and analytics.
  • Approach: Selenium scraping with fallback heuristics, metadata standardization, Airflow DAG refresh.
  • Impact: Reduced manual collection and created a reusable ingestion pattern.

Tools: Python, Selenium, Airflow, GCS

IMDb BI Analytics

Media warehouse and BI layer over 14GB of IMDb data.

  • Objective: Dimensional warehouse + dashboards for content and rating insights.
  • Approach: Alteryx cleaning, Snowflake star schema, ADF pipelines with deduplication/aggregation rules.
  • Impact: Scalable exploration via Power BI dashboards (genres, ratings, crew).

Tools: Alteryx, Azure Data Factory, Snowflake, Power BI, SQL

Chicago & Dallas Food Inspections

Multi-city food safety analytics over standardized inspection data.

  • Objective: Standardize and compare inspection risk/compliance across cities.
  • Approach: Normalize Dallas wide schema to align with Chicago, Databricks Bronze→Silver, Snowflake + Tableau analytics.
  • Impact: Unified multi-city dataset powering high-risk and hotspot KPIs.

Tools: Databricks, Snowflake, Tableau, SQL, Azure Data Factory

NYPD Arrest Analytics

Refreshable analytics over 5M+ arrest records.

  • Objective: Repeatable analysis across demographics, offenses, place, and time.
  • Approach: Snowflake modeling, incremental MERGE-based fact table, Power BI decision dashboards.
  • Impact: Shifted from static reporting to refreshable analytics.

Tools: Snowflake, Power BI, SQL, Alteryx, Azure Data Factory

Export Growth Analysis

Econometric modeling for trade and tariff insights.

  • Objective: Quantify relationships between tariffs, macro factors, and exports.
  • Approach: Python cleaning/joining, GLM/Poisson regression, reproducible analysis notebooks.
  • Impact: Interpretable, model-driven insights into export drivers.

Tools: Python, Pandas, GLM/Poisson regression
